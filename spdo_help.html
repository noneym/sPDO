Giriş

Spdo nedir?

Spdo bir veritabanı sınıfıdır. MySQL üzerinde CRUD işlemlerini kolaylaştırmak, geliştiriciyi PHP'nin eski mysql fonksiyonlarından kurtarmak ve geliştiriciyi nesne tabanlı programlamaya teşvik etmek amaçlarını gütmektedir. Spdo sınıfı, arkaplanda PHP'nin PDO eklentisini kullanmaktadır.

Geçmiş

Başlangıç sürümü, PDO eklentisini daha kolay ve güvenli bir şekilde kullanmak, PDO fonksiyonlarını Spdo sınıfı içerisinde isteğe bağlı kullanabilme imkanı tanımak ve bu fonksiyonları static yapıda kullanabilmek için hazırlanmıştı.

1.1.0 sürümünde; çekirdek düzenlemesi yapılarak fonksiyonlar public yapıya çevrildi, PDO eklentisine ait fonksiyonların kullanımı kaldırıldı ve sorgu sonuçlarını önbellekleme özelliği eklendi.

Neler yeni?

  • Çekirdek ve fonksiyonlar, işlemlerin olası hatalarda daha hızlı sonuca erişmesi ve iç içe if else bloklarının sebep olduğu kod yığınından kurtulması için istisna yönetimi try catch yapısına geçildi.
  • insert(), update() ve delete() fonksiyonları execute() fonksiyonunda birleştirildi.
  • SQL sorguları kullanmadan, pratik bir şekilde sorgu çalıştıran fonksiyonlar eklendi.
    • select()
    • where()
    • order()
    • group()
    • limit()
    • result()
    • insert()
    • update()
    • delete()

Gereksinimler

Spdo sınıfı, arkaplanda PDO eklentisini kullandığı için, sunucu ortamınızda PDO eklentisinin kurulu olması ve PDO için MySQL uzantısının aktif olması gerekmektedir. PDO eklentisinin kurulumu için açıklamalara PHP.net sitesinde bu linkten erişebilirsiniz.

Kurulum

Spdo sınıfını indirip, açtığınızda bu dosyalara erişeceksiniz:

- codeigniter
    - application
        - config
            | spdo.php
        - libraries
            | Spdo.php
        - third_party
            - sPDO
                + cache
                | core.php
- other
    + cache
    | config.php
    | Spdo.php
    | Spdo_Core.php
| spdo_help.html

Spdo sınıfını hangi platformda kullanmak istiyorsanız aşağıdaki başlıklardan size uygun olanını seçip kurulum adımları takip etmelisiniz.

Genel Kullanım

İndirdiğiniz zip dosyasını açtığınızda codeigniter ve other klasörleri ile karşılaşacaksınız. Genel kullanım için other klasöründeki dosyaları kullanacaksınız.

  • Spdo.php ve Spdo_Core.php dosyalarını scriptinizin bulunduğu dizine kopyalayın.
  • Bir ayar dosyası oluşturun veya config.php dosyasını scriptinizin bulunduğu dizine kopyalayın. Eğer ayar dosyası oluşturmak istiyorsanız; 
    <?php
 $spdo_config = array(
     'username'      => 'MYSQL KULLANICI ADI',
     'password'      => 'MYSQL KULLANICI PAROLASI',
     'database'      => 'MYSQL VERITABANI ADI',

     # Sunucu ortamınızda özel bir yapılandırma yoksa, hostname anahtarının alacağı değer localhost kalmalı.
     'hostname'      => 'localhost',

     # Sorgulama işleminde kullanılan karakter seti. Varsayılan değer utf8'dir.
     'char_set'      => 'utf8',

     # Önbellekleme özelliği, kaynak kullanımını en aza indirmeyi amaçlar.
     # SQL sorguları tekrar sorgulandığında, belirlediğiniz süreden eski değilse diskten veriyi okur.
     # Etkinleştirmek için true, pasifleştirmek için false değerini atamalısınız.
     'caching'       => false,

     # Önbellekleme özelliği etkin ise cache_dir anahtarına atanan değer önbellek dosyalarının tutulacağı dizin olarak
     # benimsenir. Bu dizin yazılabilir olmalıdır.
     'cache_dir'     => 'cache',

     # Önbellekleme özelliği etkin ise belirlediğiniz saniye cinsinden süre boyunca diskteki sonuç dönecektir.
     'cache_expire'  => 7200,
 );
 ?>
  • Spdo.php ve Spdo_Core.php dosyalarını require_once metodu ile script dosyanıza çağırın. 
    <?php
require_once 'Spdo_Core.php';
require_once 'Spdo.php';
?>
  • Ayar dosyasını require_once metoduyla script dosyanıza çağırın. 
    <?php
require_once 'config.php';
?>
  • Sınıfı bir değişkene yükleyin. 
    <?php
$spdo = new Spdo($spdo_config);
?>

Codeigniter

Codeigniter framework'ü ile çalışıyorsanız, kurulum adımlarında gerekli olacak dosyaları codeigniter klasöründe bulacaksınız.

  • application dizini içeriğini projenizin application dizinine dahil edin.
  • config dizininde spdo.php dosyasını düzenleyin.
  • Sınıfın otomatik olarak yüklenmesini istiyorsanız, config dizininde yer alan autoload.php dosyasında şu düzenlemeyi yapın; $autoload['libraries'] dizisine sınıfın ismini küçük harfler ile değer olarak ekleyin. 
    $autoload['libraries'] = array('spdo');
    Eğer, otomatik yüklenmesini istemiyorsanız controller dosyanızda şu kodu kullanın: 
    $this->load->library('spdo');

get_results()

get_results ( string $sql_query [, mixed $bind [, string $parser ]] )

SQL sorguları ile işlem yapmak, tablonun tamamını veya bir bölümünü belirlenen ayrıştırma türünde geri döndürmek amacıyla kullanılmaktadır. Ayrıca, SQL injection açıklarından korunmak için PDO eklentisinin binding sistemini kullanmaktadır.

# SELECT * FROM users
$results = $spdo->get_results('SELECT * FROM users');
var_dump($results);

İlk örnek get_results() fonksiyonunun en sade kullanımıdır.

# SELECT * FROM users WHERE id > 70 AND id < 90
$results = $spdo->get_results('SELECT * FROM users WHERE id > ? AND id < ?', array(70,90));

İkinci örneğimizde PDO eklentisinin özelliği olan binding sistemini kullandık. İkinci parametre; string, integer veya array olabilir. Eğer, bind sisteminde birden fazla değer kullanılması gerekiyorsa, örneğimizdeki gibi ikinci parametremizde veri türü array olmalı ve anahtarlar integer türünde olmalı. Aksi halde, bind sistemi çalışmayacaktır.

# SELECT * FROM users WHERE id > 70 AND id < 90
$results = $spdo->get_results('SELECT * FROM users WHERE id > ? AND id < ?', array(70,90), 'assoc');
var_dump($results);

Üçüncü örneğimizde sorgu sonuçlarını ayrıştırma türümüzü belirledik. Spdo sonuçları varsayılan olarak object tipinde döndürmektedir. Fakat, array tipinde döndürmek istiyorsak üçüncü parametreye assoc değerini girmemiz gerekmektedir.

foreach ($results as $result) {
    echo $result->id;
    echo "..." . $result->name;
    echo "..." . $result->email;
    echo "<br><br>";
}

Dönen sorgu sonuçlarını kullanmak için foreach veya while döngülerini kullanabilirsiniz.

get_row()

get_row ( string $sql_query [, mixed $bind [, string $parser ]] )

SQL sorguları ile işlem yapmak, tablodan bir satır veriyi belirlenen ayrıştırma türünde geri döndürmek amacıyla kullanılmaktadır. Ayrıca, SQL injection açıklarından korunmak için PDO eklentisinin binding sistemini kullanmaktadır.

# SELECT * FROM users WHERE id = 5
$result = $spdo->get_row('SELECT * FROM users WHERE id = 5');
var_dump($result);

İlk örnek get_row() fonksiyonunun en sade kullanımıdır.

# SELECT * FROM users WHERE id = 5
$result = $spdo->get_row('SELECT * FROM users WHERE id = ?', 5);
var_dump($result);

İkinci örneğimizde PDO eklentisinin özelliği olan binding sistemini kullandık. İkinci parametre; string, integer veya array olabilir. Eğer, bind sisteminde birden fazla değer kullanılması gerekiyorsa, ikinci parametremizde veri türü array olmalı ve anahtarlar integer olmalıdır. Aksi halde, bind sistemi çalışmayacaktır.

# SELECT * FROM users WHERE id = 5
$result = $spdo->get_row('SELECT * FROM users WHERE id = ?, 5, 'assoc');
var_dump($result);

Üçüncü örneğimizde sorgu sonuçlarını ayrıştırma türümüzü belirledik. Spdo sonuçları varsayılan olarak object tipinde döndürmektedir. Fakat, array tipinde döndürmek istiyorsak üçüncü parametreye assoc değerini girmemiz gerekmektedir.

echo "Ad: " . $result->name;
echo "<br>";
echo "Email: " . $result->email;

get_row() fonksiyonunda dönen sonuç için döngü yardımına ihtiyaç yoktur.

get_var()

get_var ( string $sql_query [, mixed $bind ] )

SQL sorguları ile işlem yapmak, sorgu sonucunda dönen veriden tek bir alana ait veriyi geri döndürmek için kullanılmaktadır. Ayrıca, SQL injection açıklarından korunmak için PDO eklentisinin binding sistemini kullanmaktadır.

# SELECT email FROM users WHERE id = 27
$email = $spdo->get_var('SELECT email FROM users WHERE id = 27');
var_dump($email);

İlk örnek get_var() fonksiyonunun en sade kullanımıdır.

# SELECT email FROM users WHERE id = 27
$email = $spdo->get_var('SELECT email FROM users WHERE id = ?', 27);
var_dump($email);

İkinci örneğimizde PDO eklentisinin özelliği olan binding sistemini kullandık. İkinci parametre; string, integer veya array olabilir. Eğer, bind sisteminde birden fazla değer kullanılması gerekiyorsa, ikinci parametremizde veri türü array olmalı ve anaharlar integer olmalı. Aksi halde, bind sistemi çalışmayacaktır.

echo $email;

get_var() fonksiyonunda dönen sonuç için döngü yardımına ihtiyaç yoktur.

execute()

execute ( string $sql_query [, mixed $bind ] )

SQL sorguları ile ekleme, güncelleme ve silme işlemleri yapmak için kullanılmaktadır. Ekleme işleminde veritabanına eklenen satır ID değerini, güncelleme ve silme işlemlerinde etkilenen satır sayısını geriye döndürür. İşlemin olumsuz sonuçlanması durumunda false döner.

# INSERT INTO users SET name = "Gürkan", email = "gurkanbicer@yandex.com"
$result = $spdo->execute('INSERT INTO users SET name = "Gürkan", email = "gurkanbicer@yandex.com"');
var_dump($result);

İlk örnek execute() fonksiyonu ile bir ekleme işlemidir.

# INSERT INTO users name = ?, email = ?
$result = $spdo->execute('INSERT INTO users SET name = ?, email = ?',array("Gürkan","gurkanbicer@yandex.com"));
var_dump($result);

İkinci örnekte PDO eklentisinin binding sistemini kullandık.

# UPDATE users SET name = "Gürkan Biçer", email = "gurkanbicer@yandex.com" WHERE id = 25
$result = $spdo->execute('UPDATE users SET name = ?, email = ? WHERE id = ?',array("Gürkan Biçer", "gurkanbicer@yandex.com", 25));
var_dump($result);

Üçüncü örnekte bir güncelleme sorgusunu execute() fonksiyonu ve binding sistemi ile kullandık.

# DELETE FROM users WHERE id = 25
$result = $spdo->execute('DELETE FROM users WHERE id = ?', 25);
var_dump($result);

Dördüncü örnekte bir silme sorgusunu execute() fonksiyonu ve binding sistemi ile kullandık.

select()

select( string $col [, string $tbl ] )

SQL sorgusu yazmadan, veri sorgulamak için kullanılmaktadır. Ancak, sonucu geri döndürmek için result() fonksiyonu ile beraber kullanılması zorunludur.

# SELECT * FROM users
$result = $spdo->select('*','users')->result();
var_dump($result);

İlk örnekte veri object tipinde dönmektedir. Eğer, array tipinde geri döndürülmek isteniyorsa result() fonksiyonun ilk parametresine assoc değerini girmeniz gerekmektedir.

# SELECT id,name FROM users
$query = $spdo->select('id,name','users');
$result = $query->result();
var_dump($result);

İkinci örnekte result() fonksiyonunun bir alt satırda kullandık. $query->result() olarak kullanılmasının sebebi select() fonksiyonunun sınıfı $query değişkenine yüklemesidir. İlk örnekteki gibi tek satırda da kullanılabilir, ikinci örnekte gibi alt alta da kullanılabilmektedir.

where()

where ( mixed $body [, mixed $bind ] )

SQL sorgusu yazmadan, veri sorgulama işleminde filtreleme görevi görmektedir. Kullanımı için select() fonksiyonuna ve veri döndürmek için result() fonksiyonuna ihtiyaç duyar.

# SELECT * FROM users WHERE id > 5 AND id < 10
$result = $spdo->select('*','users')->where(array('id > ?','id < ?'),array(5,10))->result();
var_dump($result);

veya

# SELECT * FROM users WHERE id > 5 AND id < 10
$query = $spdo->select('*','users');
$query->where(array('id > ?','id < ?'),array(5,10));
$result = $query->result();
var_dump($result);

Binding sistemini kullanmayacaksanız:

$query->where(array('id > 5','id < 10'));

group()

group ( string $col )

Belirlenen kolona ait aynı değeri içeren satırları gruplandırmak için kullanılmaktadır.

# SELECT count(*), name FROM users GROUP BY name
$query = $spdo->select('count(*), name','users');
$query->group('name');
$result = $query->result();
var_dump($result);

Örnekte dönen sonuç name değeri aynı olan kayıt sayısı ve name değerleri olacaktır.

order()

order ( string $order_by [, int $sort ] )

Belirli bir kolona göre artan veya azalan yönde sıralamak için kullanılmaktadır. İlk parametre verinin hangi kolona göre sıralanacağını belirler. Verinin artan yani "asc" olarak sıralanması gerekiyorsa ikinci parametre 1 değeri, azalan yönde "desc" olarak sıralanması gerekiyorsa -1 değerini almalıdır. İkinci parametre boş bırakılırsa, varsayılan olarak artan yönde sıralama işlemi yapar.

# SELECT * FROM users ORDER BY id DESC
$query = $spdo->select('*','users');
$query->order('id',-1);
$result = $query->result();
var_dump($result);

limit()

limit ( int $start [, int $show ] )

Belirli bir kayıt aralığındaki satırların seçilmesi için kullanılır. İlk parametre seçilecek satır aralığının başlangıç sayısıdır, ikinci parametre ise başlangıçtan itibaren seçeceği kayıt sayısıdır. Eğer ikinci parametre boş bırakılırsa, ilk parametreye yazılan değer, limit değer olarak belirlenir.

# SELECT * FROM users LIMIT 0,5
$query = $spdo->select('*','users');
$query->limit(0,5);
$result = $query->result();
var_dump($result)

result()

result ( string $parse )

select() ve diğer yardımcı fonksiyonlarla (where, group, order, limit) birlikte kullanılmaktadır. İlk parametre ayrıştırma türünü belirlemektedir. Fonksiyon, select ve yardımcı fonksiyonlarının oluşturduğu sorguyu çalıştırmak ve geri döndürmek için kullanılmaktadır.

insert()

insert ( string $tbl [, array $data ] )

Veritabanına kayıt eklemek için kullanılmaktadır. İlk parametre tablo adını, ikinci parametre tabloya eklenecek değerleri içermektedir. Dizide, anahtar isimleri kolon ismi, değerler eklenecek veriler olarak hazırlanmalıdır.

$insert_data = array(
    'name'  => 'Gürkan',
    'email' => 'gurkanbicer@yandex.com',
);
$result = $spdo->insert('users',$insert_data);
var_dump($result);

insert() fonksiyonu doğrudan binding sistemini kullanmaktadır. İşlemin gerçekleşmesi sonucunda eklenen satırın ID değerini döndürür. Olumsuz olması durumunda false döner.

update()

update ( string $tbl [, array $data [, mixed $where [, mixed $where_bind ]]] )

Veritabanında kayıt güncellemek için kullanılmaktadır. İlk parametre tablo adını, ikinci parametre verileri, üçüncü parametre filtre, dördüncü parametre ise filtre için binding sistemini kullanılır. İkinci parametreye girilen veri array tipinde olmalı ve anahtar isimleri kolon adı, değerler ise güncellenecek yeni değerler olmalıdır. Üçüncü parametre array veya string olabilir. Binding sistemi kullanılacaksa, dördüncü parametreninde kullanılması gerekmektedir.

# UPDATE users SET name = "Gürkan" WHERE id = 25
$update_data = array(
    'name' => "Gürkan",
);
$result = $spdo->update('users',$update_data,'id = ?',25);
var_dump($result);

delete()

delete ( string $tbl [, mixed $where [, mixed $where_bind ]] )

Veritabanında kayıt silmek için kullanılmaktadır. İlk parametre tablo adını, ikinci parametre filtre, üçüncü parametre ise filtre için binding sistemini kullanır.

# DELETE FROM users WHERE id = 5
$result = $spdo->delete('users','id = ?',5);
var_dump($result);
